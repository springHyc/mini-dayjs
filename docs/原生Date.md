# 知识点记录：原生 `Date` 的功能与实现原理

## 1. `Date` 在做什么

在 JavaScript 里，`Date` 主要做三件事：

- **存一个时刻**（内部是一个时间值）
- **从各种输入解析**成该时刻
- **按 UTC 或本地时区**读出年月日时分秒，或**格式化成字符串**

它和「日历业务库」不同：不负责农历、复杂排期；也不内置像 Day.js `duration` 那样的「无锚点时长」对象模型。

---

## 2. 构造与内部表示

### 2.1 常见构造方式

| 写法                                     | 含义                                                 |
| ---------------------------------------- | ---------------------------------------------------- |
| `new Date()`                             | 当前时刻                                             |
| `new Date(毫秒时间戳)`                   | 相对 1970-01-01 00:00:00 **UTC** 的毫秒数            |
| `new Date('ISO 8601 字符串')`            | 解析（边界情况多，生产环境建议优先时间戳或标准 ISO） |
| `new Date(年, 月, 日, 时, 分, 秒, 毫秒)` | 按**本地时区**解释各分量                             |

### 2.2 规范里的「时间值」

`Date` 实例内部就是一个 **UTC 毫秒时间戳**（无效时为 `NaN`）。  
**年月日时分秒**不是单独存三份字段再同步，而是**由这个时间值 + 时区规则换算出来**的。

---

## 3. 静态方法

- `Date.now()`：当前时间戳（毫秒）
- `Date.parse(string)`：把字符串解析成时间戳（与 `new Date(str)` 类似）
- `Date.UTC(year, month, ...)`：按 **UTC** 把各分量合成时间戳

---

## 4. 实例方法（分类记忆）

### 4.1 核心数值

- `getTime()`、`valueOf()`：毫秒时间戳（便于比较、运算）

### 4.2 读 UTC

- `getUTCFullYear()`、`getUTCMonth()`、`getUTCDate()`、`getUTCHours()` 等

### 4.3 读本地时区

- `getFullYear()`、`getMonth()`、`getDate()`、`getHours()` 等  
  会受**本机时区、夏令时**影响；同一 `getTime()`，在不同时区机器上本地读数可能不同。

### 4.4 修改时刻

- `setFullYear()`、`setMonth()`、`setDate()`、`setHours()` 等，以及对应的 `setUTC*`

### 4.5 转成字符串

- `toISOString()`：UTC 下的 ISO 字符串
- `toString()`、`toUTCString()` 等
- `toLocaleString()`、`toLocaleDateString()`、`toLocaleTimeString()`：依赖运行环境的 **Intl / 区域设置**（语言、时区）

---

## 5. 它不擅长什么

- **复杂日历规则**：节假日、农历等需额外数据或库。
- **「多久」作为独立模型**：规范里没有完整的 Duration 对象；相对时间（如「3 天前」）通常自己算或用 Day.js 等。
- **随意字符串解析**：非 ISO 格式兼容性差，线上尽量约定 **时间戳** 或 **标准 ISO 字符串**。

---

## 6. 实现原理（从规范到引擎）

### 6.1 规范层面

- 实例 ≈ **包了一层数字**：一个 **UTC 毫秒时间戳**（或 `NaN`）。
- `getMonth()` / `getHours()` 等：引擎用该时间戳 + **时区与夏令时规则**，算出「本地日历上的分量」；`getUTC*` 则按 UTC 切分。

### 6.2 引擎与系统

- 浏览器、Node 等（如 V8）一般会依赖操作系统或 **ICU / 时区数据库** 处理时区、夏令时、区域格式（`toLocale*`）。
- **没有魔法**：核心就是时间轴上的一个毫秒数；其余是展示与换算。

### 6.3 和 Day.js / `duration` 的对照（帮助记忆）

| 概念                           | 典型用途                                                                    |
| ------------------------------ | --------------------------------------------------------------------------- |
| `Date`                         | **一个时间点** + 系统时区下的读写与格式化                                   |
| ISO 8601 Duration（如 `P12M`） | **无锚点的时长**；年与天换算依赖上下文，和「两个时刻的 diff」不是同一类问题 |

---

## 7. 一句话总结

**`Date` = 内部一个 UTC 毫秒时间戳；本地/UTC 的「年月日时分秒」都是算出来的；底层靠引擎 + 系统/时区数据完成换算与本地化展示。**
